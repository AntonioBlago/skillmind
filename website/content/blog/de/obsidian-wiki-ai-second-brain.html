Your AI coding assistant learns things every day — workflow preferences, client context, debugging patterns, SEO strategies. But where does all that knowledge go? Into a vector database that only the AI can search. You never see it.

What if you could turn that invisible memory into a visual knowledge graph you can browse, search, and explore? That's exactly what SkillMind's new Obsidian export does.

TL;DR: SkillMind now exports all your AI assistant's memories as an Obsidian vault with [[wikilinks]], topic maps, color-coded graph groups, and full-text search. Inspired by Andrej Karpathy's LLM wiki pattern. One command: skillmind export ~/MyWiki

The Problem: Your AI Has Knowledge You Can't See

If you use Claude Code with a memory system, your assistant accumulates hundreds of memories over time: feedback on your coding style, project deadlines, client preferences, tool configurations, learned workflows. These live in a vector database (Pinecone, ChromaDB, etc.) and are retrieved via semantic search.

This works great for the AI. But for you, it's a black box:

  • You can't browse what the AI knows about you
  • You can't see connections between topics
  • You can't share the knowledge with colleagues
  • You can't use it outside of AI conversations

RAG (Retrieval Augmented Generation) solves the AI's problem. But it doesn't solve yours.

The Karpathy Wiki Pattern: Beyond RAG

In April 2026, Andrej Karpathy shared a concept that went viral: instead of using traditional RAG pipelines, let the LLM compile raw knowledge into a structured wiki. The wiki becomes a navigable, interlinked knowledge base that both humans and AI can use.

Key insight: at the scale of personal knowledge (~100 articles, ~400K words), you don't need embeddings or vector search. A well-maintained index with [[wikilinks]] is enough for the LLM to navigate efficiently — and it's infinitely more useful for the human.

SkillMind implements this pattern as a direct export from your vector database to an Obsidian vault.

How It Works: Vector DB to Obsidian in One Command

SkillMind stores memories in a vector database (Pinecone, ChromaDB, Qdrant, FAISS, or Supabase). The new Obsidian exporter reads all memories and generates a complete wiki vault:

# Export all memories to an Obsidian vault
skillmind export ~/Documents/MyWiki

# Or via MCP tool in Claude Code
export_obsidian vault_path="~/Documents/MyWiki"

What Gets Generated

MyWiki/
├── .obsidian/             # Pre-configured graph groups & bookmarks
├── CLAUDE.md              # Wiki maintenance instructions
├── raw/                   # Drop raw source material here
└── wiki/
    ├── index.md           # Master index by category and topic
    ├── log.md             # Operation history
    ├── skills/            # Skill memories (green in graph)
    ├── references/        # Reference memories (blue)
    ├── feedback/          # Feedback memories (teal)
    ├── projects/          # Project memories (orange)
    ├── users/             # User profile memories (purple)
    └── topics/            # Topic MOC pages (light blue)

Categorized Knowledge with Color-Coded Graph

Every memory is placed in a category folder based on its type. When you open the vault in Obsidian and press Ctrl+G, you see a color-coded knowledge graph:

CategoryColorWhat It Contains
SkillsGreenPatterns, workflows, how-tos, learned techniques

ProjectsOrangeClient context, deadlines, project status

The graph groups are pre-configured in .obsidian/graph.json — no manual setup needed. Just open the vault and you see your knowledge network instantly.

Features that make this work in Obsidian:

  • YAML frontmatter tags — appear in Obsidian's tag pane for filtering
  • Created/updated dates — ISO format recognized by Obsidian and Dataview
  • Aliases — alternative names for search
  • [[Wikilinks]] — cross-category connections between memories
  • Topic MOC pages — Maps of Content grouping memories by subject
  • Category index pages — overview per type with dates and tags

RAG + Wiki: Best of Both Worlds

SkillMind doesn't replace RAG — it adds a human-readable layer on top. Your memories still live in the vector database for semantic search during AI conversations. The Obsidian vault is an additional output format that makes the same knowledge accessible to you.

The sync is incremental: run skillmind sync to add only new memories without regenerating existing pages.

Getting Started in 5 Minutes

Step 1: Install SkillMind

pip install skillmind[pinecone,mcp]

Step 2: Export to Obsidian

# Full export
skillmind export ~/Documents/MyWiki

# Or via MCP in Claude Code:
# "export all my memories to an Obsidian vault"

Step 3: Open in Obsidian

  1. Download Obsidian (free, no account needed)
  2. Click "Open folder as vault"
  3. Select your export folder
  4. Press Ctrl+G for the knowledge graph

Step 4: Keep It in Sync

# Add new memories only (fast)
skillmind sync

# Full rebuild (regenerate everything)
skillmind export ~/Documents/MyWiki --full-rebuild

YouTube Learning: Build Knowledge from Videos

SkillMind can also learn from YouTube videos — extracting transcripts, structuring key insights via Claude API, and storing them as memories. These then appear in your Obsidian vault after sync:

# Learn from a video
skillmind learn-youtube "https://youtube.com/watch?v=..."

# Learn from a channel
skillmind learn-youtube-channel "UCxxxxx" --limit 5

# Sync to Obsidian
skillmind sync

If YouTube blocks direct connections, SkillMind supports ScraperAPI proxy routing — just add your API key to .env.
